# seccomp-privacy-platform · 功能与实现总览

> 一份"看完就能下载、跑、答辩"的项目档案。包含主要功能列表、实现路径、文件指引、运行方式、发布工件清单。

许可证：GPL-3.0-or-later（详见根 [LICENSE](LICENSE) 与 [NOTICE](NOTICE)）。

---

## 0. 一句话定位

**基于 PJC（Private Join and Compute）与 SSE（Searchable Symmetric Encryption）的电商隐私数据查询平台**，让商家与平台在 **不交换用户明细** 的前提下完成跨方归因、复购、客服回访、风控拼图等典型数据合作场景；原始 join key 全程经 SSE / HMAC / KMS 保护，输出仅暴露聚合结果。

主链路：

```text
SSE 候选导出 → 受控记录恢复 → Rust Bridge 令牌化 → A-PSI / PJC 求交 → 策略发布 → 审计封印
```

## 1. 主要功能（按模块）

### 1.1 SSE 可搜索对称加密存储 — `sse/`

| 能力 | 实现位置 | 备注 |
| --- | --- | --- |
| CJJ14.PiBas 加密关键字索引 | `sse/schemes/CJJ14/PiBas/` | 默认 scheme，AES-256-GCM + HMAC-SHA256 |
| 倒排索引 / 多 key 索引 | `sse/frontend/server/services/*`，`sse/example_usage.py` | 同一组 value 可绑定多个关键字 |
| 加密记录存储 | `sse/toolkit/encrypted_record_store.py` | PBKDF2HMAC-SHA256 派生密钥 + AES-256-GCM 块加密 + keyed HMAC-SHA256 记录 ID tag |
| Client CLI | `sse/run_client.py` | `create-service` / `encrypt-database` / `upload-config` / `upload-encrypted-database` / `search` / `multi-search` / `create-encrypted-record-store` / `export-bridge-records` / `serve-record-recovery[-http]` |
| Server CLI | `sse/run_server.py start` | WebSocket 服务端，HOST/PORT 经 `SSE_SERVER_*` 环境变量配置 |

**一次性 SSE 搜索助手**：`scripts/sse_oneshot_search.py` 自动完成"起服务 → create-service → generate-key → encrypt-database → upload → search → 清理"七步，emit `sse_oneshot_search/v1` JSON。

## 2. 三种"跑"的方式

### 2.1 端到端主链路

```bash
bash scripts/run_sse_bridge_pipeline.sh \
  --server-source "$PWD/sse/examples/bridge_server_records.jsonl" \
  --client-source "$PWD/sse/examples/bridge_client_records.jsonl" \
  --server-join-key-field email --client-join-key-field email \
  --client-value-field amount \
  --token-scope demo-scope --token-secret local-dev-secret \
  --job-id demo_job --out-base "$PWD/tmp/demo_run"
```

完成后浏览器访问 `http://127.0.0.1:18094/jobs`（启动 dashboard 服务后）查看 stage 时序 / 公开报告 / 审计链。

### 2.2 独立 SSE 关键字搜索

CLI 路径（手动）：

```bash
cd sse && source .venv/bin/activate
python run_server.py start &
python run_client.py generate-config --scheme CJJ14.PiBas --save-path /tmp/cfg.json
python run_client.py create-service --config /tmp/cfg.json --sname demo
python run_client.py generate-key --sname demo
python run_client.py encrypt-database --sname demo --db-path example_db.json
python run_client.py upload-config --sname demo
python run_client.py upload-encrypted-database --sname demo
python run_client.py search --sname demo --keyword China --output-format hex
```

一键路径（SPA 或 helper 都行）：

```bash
echo '{"keyword":"China","db_path":"'"$PWD/sse/example_db.json"'","output_format":"hex"}' > /tmp/req.json
python3 scripts/sse_oneshot_search.py --request-file /tmp/req.json
# 或浏览器：http://127.0.0.1:18094/sse-query
```

### 2.3 独立 PJC 求交

```bash
# 先用 bridge prepare-job 产出 server.csv + client.csv
./bridge/target/release/bridge prepare-job \
  --server-input bridge/examples/server_export.csv --server-input-format csv \
  --server-join-key-column email --server-normalizer email \
  --client-input bridge/examples/client_export.csv --client-input-format csv \
  --client-join-key-column email --client-value-column amount --client-value-mode raw-int \
  --client-normalizer email \
  --out-dir /tmp/pjc_demo --job-id pjc_demo \
  --token-scope demo --token-secret local-dev-secret

# 走 helper（或 SPA `/pjc-only`）
echo '{
  "server_csv":"/tmp/pjc_demo/server.csv",
  "client_csv":"/tmp/pjc_demo/client.csv",
  "job_meta":"/tmp/pjc_demo/job_meta.json",
  "job_id":"console-pjc-1",
  "threshold_k":1,"max_queries":5,"pjc_build":false
}' > /tmp/req.json
python3 scripts/pjc_run_only.py --request-file /tmp/req.json
```

预期：`intersection_size=2`，`intersection_sum=425`，`released=true`。

## 6. 项目结构

```text

├── bridge/                 # Rust：normalize + HMAC + job metadata
├── sse/                    # Python：SSE server/client + 加密记录存储 + 受控导出
├── services/               # Python：长期运行服务（record-recovery HTTP / Unix）

├── console/                # Operator Console SPA（Vite + React + TS）
│   ├── src/api/            # 6 个 sidecar API client
│   ├── src/components/     # 设计系统（Button / DataTable / JsonBlock / Toast / Modal / Tabs / ...）
│   └── src/routes/         # 17 个路由（含 /sse-query 和 /pjc-only）
├── scripts/                # 编排 / 契约 / benchmark / observability / KMS / dashboard / SPA helper
├── migrations/             # SQL 控制面 schema（SQLite + PostgreSQL）
├── schemas/                # 80+ JSON 契约
├── config/                 # 示例配置（policy / keyring / KMS / observability / HA / SPA manifest）

├── docs/                   # 70+ 篇深度文档
└── .github/workflows/      # CI smoke + release 流水线
```
